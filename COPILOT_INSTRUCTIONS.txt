# ══════════════════════════════════════════════════════════════════════
# IF ANY DEPLOY FAILS — DEBUGGING RULES
# ══════════════════════════════════════════════════════════════════════

# Rule 1: Read the FULL error. Modal errors are descriptive.

# Rule 2: Image build failures
#   - pip install error → check exact package name, try adding version pin
#   - git clone fails   → check network, try adding retries
#   - Never change torch or torchvision versions — exact versions are required
#     by the gsplat wheel (world2) and repo requirements

# Rule 3: Runtime failures (after image builds fine)
#   modal logs hunyuan-world2    # stream live logs
#   modal logs hunyuan-world15
#   modal logs hunyuan-world10

# Rule 4: OOM (out of memory)
#   world2:  add enable_bf16=True (already set), or reduce target_size to 448
#   world15: already using 80GB, if still OOM add --use_fp8_gemm true to cmd
#   world10: check use_fp8 checkbox in UI, or add --fp8_gemm --fp8_attention

# Rule 5: Model download fails
#   - Check HF_TOKEN secret: modal secret list
#   - Re-create: modal secret create huggingface HF_TOKEN=<token>
#   - For FLUX (world15): confirm access was approved at
#     https://huggingface.co/black-forest-labs/FLUX.1-Redux-dev

# Rule 6: Volume issues
#   - List volumes: modal volume list
#   - If weights are corrupted, delete the flag file:
#     modal volume rm world2-hf-cache /world2_downloaded.flag
#   - Container will re-download on next start

# Rule 7: DO NOT change these — they come from the repos:
#   world2:  subfolder="HY-WorldMirror-2.0"         (DOCUMENTATION.md)
#   world15: --resolution 480p                        (run.sh)
#   world15: --few_step true --num_inference_steps 4  (run.sh, distilled model)
#   world15: --model_type ar                          (run.sh)
#   world10: demo_panogen.py then demo_scenegen.py    (README)
#   world10: --classes outdoor or indoor              (README)
